## 기술적 특징

### 퀵슬롯 시스템

세이브포인트 저장 후 로드 시,
퀵슬롯에 등록된 아이템이 사라지는 문제가 발생했습니다.

이를 해결하기 위해 퀵슬롯 정보를 저장 데이터와 연동하고,
슬롯별 아이템 정보와 수량이 복원되도록 구조를 개선했습니다.

그 결과, 로드 이후에도 퀵슬롯 상태가 유지되도록 하여
플레이 진행의 연속성을 확보했습니다.

---

### 아이템 시스템 구조

아이템마다 획득 이후 처리 방식이 달라
하나의 흐름으로 관리하기 어려운 문제가 있었습니다.

이를 해결하기 위해 아이템 타입에 따라
퀵슬롯 등록, 즉시 사용, 수집 요소 등으로
처리 흐름을 구분하는 구조를 설계했습니다.

이를 통해 아이템 동작이 명확해졌고,
기능 추가 시 수정 범위를 줄일 수 있도록 개선했습니다.
